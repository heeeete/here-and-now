# 지금여기

지도 위 특정 위치에 **지금 이 순간의 현장 상황과 분위기**를 짧게 남기고, 다른 사용자가 이를 실시간으로 확인할 수 있는 서비스

---

## 1. 프로젝트 소개

**지금여기**는 정적인 리뷰 서비스가 아닙니다.

이 서비스는 아래 같은 정보를 다룹니다.

- `신림역 5번 출구 공사중`
- `당곡사거리 사고로 정체`
- `성수 골목 줄 엄청 김`
- `한강 사람 많고 분위기 좋음`

즉, 장소에 대한 장기 평가가 아니라  
**지금 그 위치에서 벌어지고 있는 일**을 공유하는 것이 핵심입니다.

---

## 2. 목표

- 사용자가 특정 장소의 현재 상태를 빠르게 확인할 수 있도록 한다
- 사용자가 현장 상황을 짧고 가볍게 제보할 수 있도록 한다
- 실시간 제보 + 분위기 공유가 결합된 위치 기반 서비스를 만든다
- 실제 사용자 반응과 반복 사용 가능성을 검증한다

---

## 3. 핵심 가치

- 리뷰보다 빠름
- 뉴스보다 가벼움
- 지도보다 현장감 있음
- 위치 정보와 짧은 텍스트를 결합해 즉시 이해 가능
- 실용 정보와 분위기 정보를 동시에 제공

---

## 4. 플랫폼

- 형태 : 반응형 웹
- 지원 : 모바일 / 데스크톱
- 초기 방향 : 지도 중심 UX
- 목적 : 모바일에서도 편하고, 데스크톱에서도 충분히 탐색 가능한 구조

---

## 5. 주요 사용자

- 20~30대 일반 사용자
- 지도를 자주 사용하는 사용자
- 번화가 / 상권 / 유동 인구 많은 지역을 자주 가는 사용자
- 약속, 이동, 산책, 데이트, 러닝 등 외부 활동이 많은 사용자
- 현장 상황이나 분위기 변화에 민감한 사용자

---

## 6. 핵심 시나리오

### 6-1. 이동 전 확인

사용자가 약속 장소 근처를 미리 확인하고 최근 제보를 본다.

### 6-2. 현장 제보

사용자가 공사, 사고, 혼잡, 행사, 분위기 같은 정보를 직접 남긴다.

### 6-3. 분위기 탐색

사용자가 특정 지역의 현재 분위기나 혼잡도를 확인한다.

### 6-4. 제보 반응

사용자가 다른 사람의 제보에 반응해 정보의 유효성을 보완한다.

## 10. UX 방향

지금여기는 **지도 맥락을 유지하는 것**이 중요합니다.

그래서 기본 원칙은 아래와 같습니다.

- 지도는 항상 메인 컨텍스트로 유지
- 작성 / 수정 / 삭제 / 상세 조회는 페이지 이동보다 오버레이 중심
- 게시판처럼 무겁게 만들지 않음
- 열람이 가장 쉬워야 하고 작성은 빠르게 끝나야 함

---

## 11. 작성 / 상세 / 수정 / 삭제 처리 방식

### 작성

- 지도에서 특정 위치 클릭
- 작성 레이어 오픈
- 카테고리 / 무드 / 코멘트 / 사진 / 비밀번호 입력
- 등록 후 현재 지도에 즉시 반영

### 상세

- 핀 클릭 시 상세 레이어 오픈
- 코멘트, 사진, 작성 시각, 남은 시간, 반응 수, 신고 버튼 표시

### 수정

- 상세 레이어에서 수정 버튼 클릭
- 비밀번호 확인
- 검증 성공 시 수정 레이어 오픈

### 삭제

- 상세 레이어에서 삭제 버튼 클릭
- 비밀번호 확인
- 검증 성공 시 삭제 처리

### 결론

초기 MVP 기준으로는  
**제보, 수정, 삭제, 상세는 페이지 라우팅보다 dialog / bottom sheet 방식이 더 자연스럽다.**

이유는 다음과 같습니다.

- 지도 상태를 유지할 수 있음
- 사용 흐름이 끊기지 않음
- 짧은 액션에 독립 페이지가 과함
- 이 서비스의 핵심은 “지도 탐색 중 바로 행동”이기 때문

## 12. 반응 기능

사용자는 제보에 아래 반응을 남길 수 있습니다.

- 도움됐어요
- 아직 유효
- 끝났어요

### 목적

- 정보가 실제로 유용했는지 판단
- 현재도 유효한 정보인지 보완
- 이미 끝난 상황인지 반영
- 허위 제보와 오래된 제보 문제를 완화

---

## 13. 만료 정책

- 모든 제보는 등록 후 6시간 뒤 자동 만료
- 만료된 제보는 기본 조회 대상에서 제외
- 오래된 정보 누적을 막고 실시간성을 유지
- 리뷰 서비스처럼 변질되는 것을 방지

---

## 14. 익명 정책

- 로그인 없이 열람 가능
- 로그인 없이 제보 가능
- 작성 시 비밀번호 입력
- 수정 / 삭제 시 해당 비밀번호로 인증

### 목적

- 참여 장벽을 낮춤
- 회원 시스템 없이 빠르게 MVP 검증 가능
- 작성자 본인만 수정 / 삭제 가능하게 제한

---

## 15. 운영 정책 방향

초기에는 넓게 허용하되, 아래는 금지합니다.

- 특정 개인 비방
- 실명 언급
- 욕설
- 혐오 표현
- 신상정보 노출
- 과도한 추측성 표현

사고, 위험, 통제 같은 제보는 허용하지만  
단정적이거나 과장된 표현은 제한하는 방향이 적절합니다.

## 18. 라우팅 방향

초기 구조는 아래처럼 가져갑니다.

### 기본 방향

- 메인 지도 화면 유지
- 작성은 dialog 또는 bottom sheet
- 상세는 dialog 또는 bottom sheet
- 수정은 비밀번호 인증 후 dialog
- 삭제는 확인 dialog

### 왜 이렇게 가는가

- 지도 컨텍스트 유지가 중요함
- 사용 흐름이 짧고 가벼움
- 별도 페이지 전환이 과함

### 예외

공유 가능한 제보 링크가 필요해지면  
제보 상세에 한해 별도 URL 진입 구조를 추후 추가할 수 있습니다.

즉 초기 방향은 아래와 같습니다.

- 평소 사용 흐름 : overlay 중심
- 추후 공유 기능 : 일부 route 추가 가능
